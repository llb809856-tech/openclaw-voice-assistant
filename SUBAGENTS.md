# 🤖 Sub-Agent 团队配置

## 架构

```
┌─────────────────────────────────────┐
│        主 Agent (小林)              │
│   模型: MiniMax-M2.5               │
│   任务: 理解需求 → 分发 → 汇总      │
└─────────────────────────────────────┘
           │
           ▼
┌─────────────────────────────────────┐
│         Sub-Agents                  │
├─────────────┬─────────────┬─────────┤
│ 选品师      │ 写手        │ 设计师   │
│ qwen3.5    │ qwen3.5    │ qwen3.5 │
├─────────────┼─────────────┼─────────┤
│ 分析师      │ 客服        │ 研究员   │
│ MiniMax    │ qwen3.5    │ qwen3.5 │
└─────────────┴─────────────┴─────────┘
```

## Sub-Agent 定义

### 1. 选品师 (ProductSelector)
- **模型**: bailian/qwen3.5-plus
- **角色**: 市场调研、竞品分析、产品筛选
- **提示词**: 你是一个专业的电商选品专家，擅长发现爆款产品...

### 2. 写手 (Copywriter)
- **模型**: bailian/qwen3.5-plus
- **角色**: 小红书文案、标题优化、商品描述
- **提示词**: 你是一个资深文案，擅长吸引人的文案写作...

### 3. 设计师 (Designer)
- **模型**: bailian/qwen3.5-plus
- **角色**: 图片处理、海报制作、视觉优化
- **提示词**: 你是一个设计师，擅长电商视觉设计...

### 4. 分析师 (Analyst)
- **模型**: minimax-cn/MiniMax-M2.5
- **角色**: 数据分析、决策建议、趋势预测
- **提示词**: 你是一个数据分析师，擅长从数据中发现 insights...

### 5. 客服 (Support)
- **模型**: bailian/qwen3.5-plus
- **角色**: 售前咨询、售后回复、问题处理
- **提示词**: 你是一个电商客服，擅长回复客户问题...

### 6. 研究员 (Researcher)
- **模型**: bailian/qwen3.5-plus
- **角色**: 行业研究、竞品监控、趋势分析
- **提示词**: 你是一个行业研究员，擅长收集和分析行业信息...

## 使用方式

```python
# Spawn 一个 Sub-agent
sessions_spawn(
    agentId="选品师",
    model="bailian/qwen3.5-plus",
    task="帮我找3款适合在抖音卖的热门产品",
    runtime="subagent"
)

# 并行 Spawn 多个
# 主 agent 分发任务给多个 sub-agent 同时执行
```

## 任务分发示例

| 场景 | 分发方式 |
|------|----------|
| 选品 + 写文案 | 并行: 选品师 + 写手 |
| 选品 + 调研 + 分析 | 并行: 选品师 + 研究员 + 分析师 |
| 单产品全流程 | 串行: 选品 → 写手 → 设计师 → 分析师 |
