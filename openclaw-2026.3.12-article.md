# OpenClaw 2026.3.12 发布：这次更新有点东西

**作者**：steipete

---

就在今天，OpenClaw 推出了 2026.3.12 版本——一次憋了很久的大更新。

说实话，这次改动比我预期的大得多，本来以为就是个常规迭代，结果 UI 重构、模型加速、插件架构调整扎堆来了，还顺手修了一波安全漏洞。

## 1. 全新控制面板

Dashboard 完全重刷了，新的 Control UI 用了自定义设计，界面焕然一新。功能分区也更清晰了：概览、聊天、配置、智能体、会话一目了然。

还加了**命令面板**（快捷键调出），移动端底部标签栏也安排上了，聊天工具也更丰富——斜杠命令、搜索、导出、置顶这些实用功能全都有。

## 2. 「快速模式」来了

这是个重磅功能：

- **OpenAI/GPT-5.4**：支持会话级快速切换，用 `/fast` 或 UI 就能搞定
- **Anthropic/Claude**：`/fast` 现在可以映射到 `service_tier` 请求，还有实时冗余验证

关键是——针对 OpenAI/Codex 请求做了专门优化，响应速度明显提升。

## 3. 插件架构大改

Ollama、vLLM、SGLang 这些模型接入方式全部迁移到了**插件架构**。以后模型发现、选择器都更标准化了，不再在核心库里硬编码，扩展性更强。

## 4. Kimi 模型终于修好了

之前 Kimi Coding 的工具调用一直接不上，这次终于解决了：

- 修复了工具调用问题，恢复原始 Anthropic 纯格式
- 现在支持 Ollama 托管思维的 Kimi 模型（如 `kimi-k2.5:cloud`）在 Thinking 模式下的工具路由
- Moonshot CN API 的 401 错误也修透了

## 5. 安全更新（划重点）

这次发版带了**一堆安全补丁**，修了多个潜在漏洞：

- **执行引擎防伪造**：转义了提示中的不可见 Unicode 字符，防止零宽字符格式化命令
- **工作区保护**：取消插件隐式自动加载，防止克隆仓库在未授权情况下跑代码
- **设备配对安全**：`/pair` 和二维码安装改为短效令牌，不再在负载中嵌入长期凭证
- **权限隔离**：沙箱子代理不能随便看父会话元数据了，`/config` 和 `/debug` 仅限业主可用

> 如果你的 OpenClaw 暴露在公网，**强烈建议立刻更新**。

## 6. 其他亮点

- **Kubernetes 支持**：新增 K8s 安装路径，提供原始 Manifests、Kind setup 及部署文档
- **Windows 更稳**：`openclaw update` 在 Windows 上不再崩溃
- **macOS 权限修复**：事项提醒权限申请问题修好了
- **Slack/ Telegram 增强**：Slack 支持 Block Kit，Telegram 优化了内联模型选择器和压缩状态显示

---

## 总结

如果你在用 **Kimi 家族模型**，或者需要在移动端更方便地管理 OpenClaw，这个版本可以直接升。

最主要的是——安全漏洞都补了，公网部署的朋友别犹豫，早升早安心。
