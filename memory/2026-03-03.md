## 🔧 MCP 技术限制（2026-03-03 15:17）

### 为什么 MCP 不能自动访问小红书官网

**Docker 容器隔离**：
- MCP 运行在 Docker 容器里
- 浏览器运行在宿主机（Mac）
- 两个环境完全隔离，无法共享 cookies

**httpOnly cookies**：
- 浏览器的登录 cookies 是 httpOnly
- JavaScript 无法获取（安全机制）
- MCP 无法通过脚本读取

**小红书安全机制**：
- 新设备需要扫码登录
- 必须用手机小红书 APP 确认
- 不能只用账号密码

### 解决方案

**共享 cookies 文件**：
1. 从浏览器数据库提取 cookies
2. 复制到 MCP 容器的 cookies.json
3. MCP 重启后使用新 cookies

**或直接用浏览器发布**：
- 已登录米米可思账号
- 不需要 MCP
- 30 秒搞定

## 🔧 子代理配置流程（2026-03-03 19:37）

### 配置步骤

**1. 编辑配置文件**：
```bash
vi /Users/a01/.openclaw/openclaw.json
```

**2. 添加子代理配置**：
```json
{
  "agents": {
    "list": [
      {
        "id": "main",
        "subagents": {
          "allowAgents": ["*"]
        }
      }
    ],
    "defaults": {
      "subagents": {
        "model": "bailian/qwen3.5-plus",
        "runTimeoutSeconds": 300
      }
    }
  }
}
```

**3. 重启网关**：
```bash
openclaw gateway restart
```

### 配置说明

| 配置项 | 说明 | 必填 |
|--------|------|------|
| `allowAgents: ["*"]` | 允许所有子代理 | ✅ |
| `subagents.model` | 子代理默认模型 | ⚠️ 建议 |
| `runTimeoutSeconds` | 超时时间（秒） | ⚠️ 建议 |

### 优缺点

**优点**：
- ✅ 飞书 bot 可以直接用
- ✅ 多任务并行
- ✅ 隔离执行

**缺点**：
- ❌ 需要修改配置
- ❌ 需要重启网关
- ❌ 成本更高（多个会话）
- ❌ 调试更困难
