## ⚠️ Kling AI 测试失败教训

**问题 1**：第一次生成视频内容不对（男人在写字，不是手套）

**原因 1**：
- `kling-video-o1` 是 Omni 模型，主要用于 **image-to-video**（图生视频）
- 我没有传 `image_list` 参考图
- prompt 没有用 `<<<image_1>>>` 格式引用图片

**问题 2**：第二次图生视频测试失败（400 错误）

**原因 2**：
- Kling 要求图片必须是 **可访问的 HTTP URL** 或 **base64 格式**
- 本地文件路径 (`/Users/a01/Desktop/...`) 不被接受
- 错误信息：`File is not in a valid base64 format`

**解决方案**：
1. 上传产品图片到图床（阿里云 OSS/腾讯云 COS/Imgur）
2. 或者先上传到 Kling 平台获取 CDN URL
3. 或者用 base64 格式传图（但数据量大可能不稳定）

**下次注意**：
- Omni 模型必须传 image_list（HTTP URL 或 base64）
- prompt 里用 `<<<image_1>>>` 引用图片
- 本地文件需要先上传到云端

## 📋 Kling API 正确用法总结

**端点**：`POST https://api-beijing.klingai.com/v1/videos/omni-video`

**认证**：
```python
JWT Token (HS256)
payload: {"iss": ACCESS_KEY, "exp": timestamp+1800, "nbf": timestamp-5}
headers: {"alg": "HS256", "typ": "JWT"}
Authorization: Bearer <token>
```

**请求体（图生视频）**：
```json
{
  "model_name": "kling-video-o1",
  "prompt": "<<<image_1>>> 描述动作",
  "image_list": [{"image_url": "纯 base64 字符串（无前缀）"}],
  "duration": "5",
  "mode": "pro",
  "aspect_ratio": "16:9"
}
```

**查询状态**：`GET /v1/videos/omni-video/{task_id}`

**成本**：约 4 credits/5 秒（¥0.5-1 元）
